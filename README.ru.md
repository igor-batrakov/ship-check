# ship-check

[English](README.md) | **Русский** | [中文](README.zh.md) | [Español](README.es.md) | [Português](README.pt.md)

**Аудит безопасности перед запуском для приложений, собранных на вайбе.** Одна команда — `/ship-check` — находит дыры,
из-за которых приходят неожиданные счета на $200, заводятся спам-боты и падают претензии от юристов, а затем проводит вас через
исправления по одному. Отвечает на вашем языке.

> Сделано для тех, кто только начал. Вы пишете ему простыми словами, он проверяет, объясняет
> каждый риск через то, во сколько он вам обойдётся, и чинит всё с вашего согласия. Ничего
> дополнительно ставить не нужно, команды запоминать тоже.

## Быстрый старт

В корне вашего проекта выполните:

```
/ship-check
```

Он определяет ваш стек, прогоняет аудит, пишет отчёт `PROD-AUDIT.md` и предлагает починить
проблемы вместе с вами — по одному изменению за раз, с вашего согласия.

## Как выглядит отчёт

`/ship-check` пишет `PROD-AUDIT.md`: сверху вердикт, а ниже то, что ударит вас первым:

```
## Вердикт: 🚫 НЕ ГОТОВО К ЗАПУСКУ

Выкатите это сегодня — и три из этих дыр ударят по вам в первый же день.

## 🔥 Чините в первую очередь
🔴 Ваш ключ OpenAI лежит в браузерном бандле (app/page.tsx:5) — любой откроет
   DevTools и потратит ваш кредит.
🔴 Row Level Security выключен на `orders` — прямо сейчас любой посетитель может
   прочитать заказы всех клиентов.
🔴 /api/chat дёргает платный API без входа и без ограничения частоты — один бот за
   ночь — это счёт на $200.

## Доступ к данным
🟢 Бакет с загрузками приватный, отдаётся через подписанные URL — проверено, чисто.
🟡 Не удалось подтвердить RLS в живой БД по коду — ваши миграции его включают;
   откройте дашборд Supabase и перепроверьте.
```

Каждая проверка приходит к одному из трёх состояний:

- 🔴 **найдена проблема** — конкретная дыра, с файлом и строкой.
- 🟢 **проверено — чисто** — проверка прошла, и в коде всё в порядке.
- 🟡 **не удалось проверить** — посмотрите вручную (нет нужного инструмента, требуется живой URL или это
  проверка поведения). 🟡 так и остаётся 🟡; аудит подскажет, на что взглянуть, вместо того чтобы гадать.

Это честное третье состояние и есть весь смысл: инструмент, который рисует ✅, будучи слепым, хуже, чем вообще никакого инструмента.

## Пример сессии

```
> /ship-check

Вижу приложение на Next.js + Supabase. Запускаю аудит перед запуском — примерно минута.
…
Вердикт: 🚫 не готово к запуску. Три вещи ударят по вам в первый же день. Починим их вместе?

> да

Первая — самая страшная: ваш ключ OpenAI в браузерном бандле, любой может его прочитать и
потратить ваш кредит. Я перенесу вызов в серверный маршрут и буду читать ключ из серверной
переменной. Применить?

> давай

Готово — ключ убран с клиента, и страница по-прежнему собирается. Риск счёта на $200 закрыт.
Дальше: Row Level Security на `orders`…
```

## Что он проверяет

Пять областей плюс соответствие требованиям — тот самый исходный чеклист вайб-кодера, с парой типичных дополнительных дыр:

- **🔑 Секреты и утечки** — ключи во фронтенде, секреты в `.env`/git/логах, API, которые отдают слишком
  много, внутренние ошибки, показанные пользователю.
- **🗄️ Доступ к данным** — Row Level Security (RLS), доступ к чужим строкам (IDOR), сервисные ключи,
  обходящие ваши проверки, файловое хранилище, по умолчанию открытое всем.
- **🔐 Надёжность авторизации** — блокировка при неверном пароле, сброс для несуществующего email,
  повторная регистрация (чеклист, который вы прокликиваете в браузере).
- **🛡️ Веб-безопасность (OWASP)** — заголовки, SQL-инъекции, XSS, валидация на сервере, debug-режим,
  забытый включённым в продакшене.
- **💸 Злоупотребления и расходы** — незащищённые вызовы платных API (риск «$200 за ночь»), ограничение
  частоты запросов, CAPTCHA на формах, CORS.
- **⚖️ Соответствие требованиям** — политика конфиденциальности, GDPR/CCPA, где живут ваши данные, с
  готовым шаблоном для старта.

## Требования

Нужен только Claude Code — аудит работает на встроенных инструментах и не требует дополнительной
настройки. Более глубокие проверки опциональны и предлагаются по ходу дела: когда мог бы помочь
инструмент вроде `gitleaks` (сканирует историю git на утёкшие ключи) или `semgrep` (шире покрывает
инъекции/XSS), `/ship-check` предложит настроить его за вас, с вашего согласия. Откажетесь — и эти
проверки останутся 🟡 «не удалось проверить». MCP вроде context7, если он у вас есть, делает советы по
исправлению точнее; он никогда не обязателен. Работает на macOS, Windows и Linux — на Windows
опциональные сканеры ставятся через `winget`/`scoop` вместо `brew`, и `/ship-check` сам всё это делает
за вас.

## Установка

Из Claude Code:

```
/plugin marketplace add igor-batrakov/ship-check
/plugin install ship-check@ship-check-tools
```

Затем запустите `/ship-check` в любом проекте. (Держите свой форк? Укажите в первой команде свой репозиторий.)

## Границы

- **Сервер вне зоны охвата.** Файрвол, база данных, открытая в интернет, запуск от root,
  SSH — это уровень сервера; его покрывает навык `new-vps-setup`.
- **Это отправная точка по соответствию требованиям, остальное — за юристом.** Проход по соответствию
  даёт чеклист и черновик политики конфиденциальности с пометкой «дайте юристу проверить».
- **Это базовый минимум, а вглубь — пентест.** Аудит ловит типичные дыры перед запуском. Серьёзному
  приложению всё равно нужен настоящий аудит безопасности.

## Для разработчиков плагина

- `skills/production-audit/SKILL.md` — оркестратор (мозг).
- `agents/*.md` — пять тонких агентов-аудиторов; логика обнаружения живёт в `references/`.
- `skills/production-audit/references/` — подробные чеклисты по каждой области (единый источник истины).
- `tests/fixtures/` и `tests/synthetic/` — намеренно уязвимые и в основном безопасные приложения;
  `tests/RESULTS.md` и `tests/SYNTHETIC.md` фиксируют, как аудит на них отрабатывает (контроль качества).

Внутренности плагина написаны на английском; отчёт и сам разговор выходят на языке
пользователя во время работы.

## Как поучаствовать

Issues и pull request'ы приветствуются — особенно про пропущенную проверку, ложное срабатывание или
новый шаблон исправления для стека, который ещё не покрыт в references. Откройте issue с кодом, на котором сработало.
